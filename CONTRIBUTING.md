# FinRisk Pro 贡献者协议

## 欢迎贡献者！

感谢您对FinRisk Pro项目的兴趣！我们欢迎各种形式的贡献，包括但不限于：
- 代码贡献
- 文档改进
- Bug报告
- 功能建议
- 测试帮助
- 翻译工作

## 行为准则

所有贡献者必须遵守我们的行为准则：
1. **尊重他人**: 保持专业和尊重的态度
2. **包容性**: 欢迎各种背景的贡献者
3. **建设性**: 提供有建设性的反馈
4. **专业性**: 保持专业的技术讨论

## 贡献流程

### 1. 开始之前
- 阅读项目文档
- 查看现有Issue和PR
- 确保环境可以正常运行项目

### 2. 创建Issue
- 描述清晰的问题或功能
- 提供复现步骤（如果是Bug）
- 标注合适的标签

### 3. 开发流程
1. Fork项目仓库
2. 创建功能分支 (`git checkout -b feature/AmazingFeature`)
3. 提交更改 (`git commit -m 'Add some AmazingFeature'`)
4. 推送到分支 (`git push origin feature/AmazingFeature`)
5. 创建Pull Request

### 4. 代码规范
- 遵循PEP 8编码规范
- 添加适当的注释和文档字符串
- 编写单元测试
- 确保所有测试通过

### 5. Pull Request要求
- 清晰描述更改内容
- 关联相关Issue
- 通过所有CI检查
- 获得至少一个核心贡献者的批准
